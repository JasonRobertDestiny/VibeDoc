# 📸 截图准备清单

为了完成阮一峰老师周刊的投稿，你需要准备以下截图：

## 必需截图（7张）

### 1. main-interface.png - 主界面
**内容要求**：
- 展示干净的主界面
- 包含输入框和按钮
- 显示项目 Logo 和标题
- 分辨率：1920x1080 或 1440x900

**拍摄步骤**：
1. 打开应用首页
2. 不要输入任何内容（保持空白状态）
3. 截取完整界面

---

### 2. generation-process.png - 生成过程
**内容要求**：
- 展示 AI 正在生成内容的实时画面
- 可以看到流式输出的文字
- 显示进度提示
- 分辨率：1920x1080

**拍摄步骤**：
1. 输入一个示例创意（比如：「一个智能健身追踪 APP」）
2. 点击生成按钮
3. 在生成过程中截图（大约 20-30 秒时）

---

### 3. development-plan.png - 完整开发计划
**内容要求**：
- 展示生成的完整开发计划
- 要能看到结构化的内容（标题、段落、列表）
- 包含项目概述、技术栈等关键部分
- 分辨率：1920x1080（可以是长截图）

**拍摄步骤**：
1. 等待生成完成
2. 滚动查看完整内容
3. 使用长截图工具（如 Snipaste、FastStone）

---

### 4. architecture-diagram.png - 架构图
**内容要求**：
- 展示自动生成的 Mermaid 架构图
- 图表要清晰可见
- 包含节点和连接线
- 分辨率：1200x800

**拍摄步骤**：
1. 在生成结果中找到架构图部分
2. 单独截取图表区域
3. 确保图表完整且清晰

---

### 5. ai-prompts.png - AI编程提示词
**内容要求**：
- 展示生成的 AI 编程提示词部分
- 要能看到专业的提示词格式
- 包含代码块或格式化文本
- 分辨率：1920x1080

**拍摄步骤**：
1. 找到「AI 编程提示词」部分
2. 截取 2-3 个提示词示例
3. 确保文字清晰可读

---

### 6. export-formats.png - 多格式导出
**内容要求**：
- 展示导出按钮或导出选项
- 可以显示 Markdown、Word、PDF、HTML 四个选项
- 分辨率：1200x600

**拍摄步骤**：
1. 生成完成后，找到导出区域
2. 截取导出按钮和选项
3. 或者展示已导出的文件列表

---

### 7. dark-mode.png - 暗色模式
**内容要求**：
- 同样的界面，但使用暗色主题
- 展示暗色模式的视觉效果
- 分辨率：1920x1080

**拍摄步骤**：
1. 切换到暗色模式（如果 Gradio 支持）
2. 或者调整系统主题
3. 截取同样的界面

---

## 可选截图（增强效果）

### 8. mobile-view.png - 移动端适配
- 展示在手机或平板上的界面
- 可以使用浏览器开发者工具模拟

### 9. export-result.png - 导出文件效果
- 打开导出的 Word 或 PDF 文件
- 展示专业的文档排版

## 📝 截图规范

### 质量要求
- ✅ 分辨率：至少 1200x800
- ✅ 格式：PNG（保证清晰度）
- ✅ 文件大小：单张 < 2MB
- ✅ 画质：无压缩、无模糊

### 内容要求
- ✅ 界面干净整洁
- ✅ 文字清晰可读
- ✅ 色彩饱和自然
- ✅ 无敏感信息（API Key 等）

### 命名规范
严格按照以下名称保存：
```
main-interface.png
generation-process.png
development-plan.png
architecture-diagram.png
ai-prompts.png
export-formats.png
dark-mode.png
```

## 📤 上传步骤

### 方法1：直接放到 GitHub（推荐）

```bash

# 然后提交到 GitHub
git add docs/images/*.png
git commit -m "docs: add screenshots for weekly submission"
git push origin master
```

然后在 WEEKLY_SUBMISSION.md 中的图片链接就能正常显示了。
